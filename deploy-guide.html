🎯 추천: GitHub Pages로 배포하기

1 GitHub 계정 생성

  1. 1. github.com 방문
  2. 2. "Sign up" 클릭하여 무료 계정 생성
  3. 3. 이메일 인증 완료

2 새 저장소 만들기

  1. 1. GitHub에 로그인 후 우측 상단 "+" → "New repository" 클릭
  2. 2. Repository name: insurance-calculator (원하는 이름)
  3. 3. Public 선택 (무료 배포를 위해 필수)
  4. 4. "Create repository" 클릭

💡 Tip: Repository 이름이 URL에 포함됩니다
예: https://yourusername.github.io/insurance-calculator/

3 파일 업로드 (웹 인터페이스 사용)

가장 쉬운 방법 (Git 명령어 불필요):

  1. 1. 생성된 저장소 페이지에서 "uploading an existing file" 클릭
  2. 2. 보험사_시책_계산기_A4가로_최종 (7).html 파일을 드래그 앤 드롭
  3. 3. 파일 이름을 index.html로 변경
  4. 4. 아래 "Commit changes" 클릭

4 GitHub Pages 활성화

  1. 1. 저장소 페이지 상단 "Settings" 클릭
  2. 2. 왼쪽 메뉴에서 "Pages" 클릭
  3. 3. Source: "Deploy from a branch" 선택
  4. 4. Branch: "main" 선택, folder: "/ (root)" 선택
  5. 5. "Save" 클릭
  6. 6. 1-2분 대기 후 페이지 새로고침
  7. 7. 상단에 표시되는 URL이 공유 링크입니다!

✅ 완료! 이제 URL을 팀원들에게 공유하세요
예: https://yourusername.github.io/insurance-calculator/

💻 고급 사용자: Git 명령어로 배포

터미널/명령 프롬프트에서 실행:

초기 설정 (최초 1회)

# Git 초기화
git init

# 파일 추가
git add index.html README.md .gitignore

# 첫 커밋
git commit -m "Initial commit: 보험사 시책 계산기"

# GitHub 저장소 연결 (YOUR_USERNAME와 YOUR_REPO를 실제 값으로 변경)
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git

# 메인 브랜치로 푸시
git branch -M main
git push -u origin main

매주/매달 업데이트

# 수정된 파일 저장
git add index.html

# 커밋 메시지 작성
git commit -m "2026년 1월 2주차 데이터 업데이트"

# GitHub에 푸시 (자동 배포)
git push

💡 자동 배포: push 후 1-2분이면 웹사이트에 반영됩니다!

🔄 매주/매달 데이터 업데이트 방법

방법 1: 웹에서 직접 수정 (추천)

  1. 1. 배포된 웹사이트에서 "⚙️ 관리자 페이지" 클릭
  2. 2. 비밀번호 입력 (기본: admin1234)
  3. 3. **인별 데이터 관리** 탭에서 엑셀 데이터 붙여넣기
  4. 4. "💾 전체 저장" 클릭
  5. 5. 브라우저 localStorage에 자동 저장됨 ✅
장점: 코드 수정 없이 데이터만 업데이트 가능!

방법 2: GitHub에서 파일 수정

  1. 1. GitHub 저장소 페이지에서 index.html 클릭
  2. 2. 연필 아이콘 (Edit) 클릭
  3. 3. JavaScript 데이터 섹션 찾아서 수정
  4. 4. "Commit changes" 클릭
  5. 5. 1-2분 후 자동 배포 완료 ✅

⚠️ 중요: 데이터 백업

브라우저 localStorage는 브라우저를 삭제하거나 캐시를 지우면 사라질 수 있습니다!

정기적으로 백업하세요:

  • 1. 관리자 페이지 → "📥 시책 데이터 다운로드"
  • 2. 관리자 페이지 → "📥 인별 데이터 다운로드"
  • 3. JSON 파일을 안전한 곳에 보관
  • 4. 필요시 "📤 데이터 불러오기"로 복원

❓ 자주 묻는 질문 (FAQ)

Q. 배포 비용이 얼마나 드나요?

A. 완전 무료입니다! GitHub Pages, Cloudflare Pages, Netlify 모두 무료 플랜으로 충분합니다.

Q. 동시 접속자 제한이 있나요?

A. GitHub Pages는 무제한 방문자를 지원합니다. 회사 전체가 사용해도 문제없습니다.

Q. 업데이트 후 얼마나 빨리 반영되나요?

A. GitHub에 push 후 보통 1-2분 이내에 자동으로 배포됩니다.

Q. 도메인을 변경할 수 있나요?

A. 네! GitHub Pages Settings에서 Custom domain을 설정할 수 있습니다. (예: calculator.yourcompany.com)

Q. 데이터가 안전한가요?

A. 데이터는 각 사용자의 브라우저 localStorage에 저장됩니다. 서버에는 전송되지 않으므로 안전합니다. 단, 정기적으로 백업하세요!

Q. Git을 몰라도 사용할 수 있나요?

A. 네! GitHub 웹 인터페이스에서 드래그 앤 드롭으로 파일을 업로드하고, 관리자 페이지에서 데이터를 수정하면 됩니다.
